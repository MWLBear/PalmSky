# PalmSky Roadmap

## 版本 2: 大道补全

- 目标: 补齐修仙流程的最终玩法闭环，让突破链路完整收束。
- 核心内容: 最终小游戏「冲九霄」。
- 附带内容: 修复历史遗留 Bug，清理版本 1 中影响体验的逻辑问题。
- 版本定位: 补齐“修仙四部曲”，让主线从修炼、突破、渡劫走向完整收官。

## 版本 3: 神游太虚

- 目标: 提升玩家留存、安全感与回归体验，降低“丢档”和“离线无反馈”的焦虑。
- 核心内容 A: 睡眠系统。
- 核心内容 B: iCloud 云备份 / 恢复。
- 设计定位: 留存与安全感。
- 版本原则: 不是实时云同步，而是“本地优先 + 云端兜底恢复”。

## 版本 4: 两仪同修

- 目标: 为双端玩家提供明确、可控的进度校准能力，降低手机与手表进度分叉带来的困扰。
- 核心内容: 手动数据覆盖系统。
- 设计定位: 双端校准 + 安全纠偏。
- 设计要求:
  - 支持“手表覆盖手机”与“手机覆盖手表”两种明确方向。
  - 覆盖目标应写入目标端本地主存档，而不是只做临时展示同步。
  - 护身符库存保持双端独立，不参与跨端覆盖。
  - 设备设置保持本地独立，不参与跨端覆盖。
  - 覆盖应采用玩家主动触发 + 二次确认，不做静默自动替换。

## 版本 5: 仙家法宝

- 目标: 在不破坏核心修炼循环的前提下增强个性化与商业化承载能力。
- 核心内容 A: 太极皮肤包。
- 核心内容 B: 周卡 / 月卡。
- 版本定位: 个性化 + 商业化。
- 设计要求:
  - 皮肤系统应独立于核心数值系统。
  - 周卡 / 月卡应优先提供体验增强，不直接破坏平衡。
  - 商业化能力需兼容 Watch 端与 iPhone 端的展示与说明逻辑。
